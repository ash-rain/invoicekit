# x402 Payment Protocol — InvoiceKit Integration

InvoiceKit exposes payment-gated API endpoints using the [x402 protocol](https://x402.org). Clients (including AI agents) pay micro-amounts of USDC on Base to access these endpoints — no accounts, no API keys, no subscriptions.

---

## How x402 Works

```
Client                   InvoiceKit Server          Coinbase Facilitator
  |                            |                            |
  |-- GET /api/x402/ping ----->|                            |
  |<-- 402 PAYMENT-REQUIRED ---|                            |
  |    (payment requirements)  |                            |
  |                            |                            |
  |-- GET /api/x402/ping ----->|                            |
  |   PAYMENT-SIGNATURE: ...   |-- POST /verify ----------->|
  |                            |<-- { isValid: true } ------|
  |                            |-- POST /settle ----------->|
  |<-- 200 OK (+ PAYMENT-RESPONSE header) --- tx hash ------|
```

1. Client requests a protected endpoint — server responds **HTTP 402** with payment requirements in the `PAYMENT-REQUIRED` header.
2. Client signs a USDC authorization (no on-chain tx yet) and retries with the `PAYMENT-SIGNATURE` header.
3. Server sends the signed authorization to the **Coinbase Facilitator** to verify and settle on Base.
4. Server returns the response with a `PAYMENT-RESPONSE` header containing the transaction hash.

---

## Endpoints

| Endpoint | Cost | Auth |
|---|---|---|
| `GET /api/x402` | Free | None — service discovery |
| `GET /api/x402/ping` | $0.001 USDC | x402 payment |
| `GET /api/x402/invoice/{number}` | $0.001 USDC | x402 payment + `X-Portal-Token` header |

All prices are set in `.env` (`X402_PRICE_USD`) and default to **$0.001 per call**.

## Wallet Setup

### 1. Create a Wallet

You need an EVM-compatible wallet to receive USDC payments on Base.

**Option A — Coinbase Wallet (recommended)**
1. Install [Coinbase Wallet](https://www.coinbase.com/wallet) (browser extension or mobile)
2. Create a new wallet, back up your seed phrase securely
3. Copy your wallet address (starts with `0x`)

**Option B — MetaMask**
1. Install [MetaMask](https://metamask.io)
2. Create or import a wallet
3. Add the **Base** network:
   - Network name: `Base`
   - RPC URL: `https://mainnet.base.org`
   - Chain ID: `8453`
   - Currency: `ETH`
   - Block Explorer: `https://basescan.org`
4. Copy your wallet address

**Option C — CDP Wallet (programmatic)**
```bash
# Coinbase Developer Platform SDK
npm install @coinbase/coinbase-sdk
```
```javascript
import { Coinbase, Wallet } from '@coinbase/coinbase-sdk';
const wallet = await Wallet.create({ networkId: 'base-mainnet' });
```

### 3. Configure InvoiceKit

Add these to your `.env` file:

```env
# Your wallet address (receives USDC payments)
X402_PAY_TO=0xYourWalletAddressHere

# Network: use testnet for development, mainnet for production
X402_NETWORK=eip155:84532           # Base Sepolia (testnet)
# X402_NETWORK=eip155:8453          # Base mainnet (production)

# USDC contract address (must match the network above)
X402_ASSET=0x036CbD53842c5426634e7929541eC2318f3dCF7e    # Base Sepolia
# X402_ASSET=0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913  # Base mainnet

# Coinbase's public facilitator (handles verification and on-chain settlement)
X402_FACILITATOR_URL=https://x402.org/facilitator

# Price per API call in USD (0.001 = $0.001 = 1000 USDC atomic units)
X402_PRICE_USD=0.001

# How long a payment authorization remains valid (seconds)
X402_MAX_TIMEOUT_SECONDS=300
```

Then run:
```bash
php artisan config:cache
```

## Protocol Details

### HTTP 402 Response

When a client hits a gated endpoint without payment, InvoiceKit returns:

```http
HTTP/1.1 402 Payment Required
Content-Type: application/json
PAYMENT-REQUIRED: <base64-encoded JSON>
```

The `PAYMENT-REQUIRED` header decodes to:

```json
{
  "x402Version": 2,
  "error": "Payment required",
  "resource": {
    "url": "https://invoicekit.app/api/x402/ping",
    "description": "InvoiceKit API access",
    "mimeType": "application/json"
  },
  "accepts": [
    {
      "scheme": "exact",
      "network": "eip155:84532",
      "amount": "1000",
      "asset": "0x036CbD53842c5426634e7929541eC2318f3dCF7e",
      "payTo": "0xYourWalletAddress",
      "maxTimeoutSeconds": 300,
      "resource": "https://invoicekit.app/api/x402/ping",
      "description": "InvoiceKit API access",
      "mimeType": "application/json",
      "extra": {
        "assetTransferMethod": "eip3009",
        "name": "USDC",
        "version": "2"
      }
    }
  ],
  "extensions": null
}
```

### Payment Header (Client → Server)

The client retries with a signed EIP-3009 authorization:

```http
GET /api/x402/ping HTTP/1.1
PAYMENT-SIGNATURE: <base64-encoded JSON>
```

The `PAYMENT-SIGNATURE` header decodes to:

```json
{
  "x402Version": 2,
  "scheme": "exact",
  "network": "eip155:84532",
  "payload": {
    "signature": "0x2d6a7588...",
    "authorization": {
      "from": "0xClientWallet",
      "to": "0xYourWallet",
      "value": "1000",
      "validAfter": "1740672089",
      "validBefore": "1740672389",
      "nonce": "0xf3746613..."
    }
  }
}
```

### Success Response

```http
HTTP/1.1 200 OK
PAYMENT-RESPONSE: <base64-encoded JSON>
Content-Type: application/json
```

The `PAYMENT-RESPONSE` header decodes to the settlement receipt:

```json
{
  "success": true,
  "transaction": "0xabc123...",
  "network": "eip155:84532",
  "payer": "0xClientWallet",
  "amount": "1000"
}
```

## Calling x402 Endpoints

### JavaScript / TypeScript

```bash
npm install @x402/core @x402/evm viem
```

```typescript
import { createX402Client } from '@x402/core';
import { privateKeyToAccount } from 'viem/accounts';

const account = privateKeyToAccount('0xYourPrivateKey');

const response = await createX402Client({
  account,
  facilitatorUrl: 'https://x402.org/facilitator',
}).get('https://invoicekit.app/api/x402/ping');

console.log(await response.json());
```

### Python

```bash
pip install x402
```

```python
from x402.client import create_client

client = create_client(
    private_key=os.getenv("EVM_PRIVATE_KEY"),
    facilitator_url="https://x402.org/facilitator",
)

response = client.get("https://invoicekit.app/api/x402/ping")
print(response.json())
```

### cURL (manual — for debugging)

First, get the 402 requirements:
```bash
curl -i https://invoicekit.app/api/x402/ping
```

Decode the `PAYMENT-REQUIRED` header:
```bash
echo "<base64-value>" | base64 -d | jq .
```

You'll need an x402-capable client to sign and retry. Use the JS/Python SDK for anything beyond debugging.

## Accessing Invoice Data

The `/api/x402/invoice/{number}` endpoint requires both:
1. An x402 payment (`PAYMENT-SIGNATURE` header)
2. A portal token (`X-Portal-Token` header) — generated from the InvoiceKit UI

**Generate a portal token:**
1. Open an invoice in InvoiceKit
2. Click **Share** → **Generate Portal Link**
3. Copy the token from the URL: `https://invoicekit.app/portal/<TOKEN>`

**Call the endpoint:**
```bash
# With the x402 Python client
response = client.get(
    "https://invoicekit.app/api/x402/invoice/INV-2024-001",
    headers={"X-Portal-Token": "your-portal-token"},
)
```

## Facilitator

InvoiceKit uses the **Coinbase public facilitator** at `https://x402.org/facilitator`. This service:
- Verifies that the client's signed authorization is cryptographically valid
- Checks USDC balance and allowance on-chain
- Executes the on-chain transfer and returns the transaction hash

The facilitator cannot move funds beyond what the client signed. Settlement is atomic — funds move only when the authorization is valid and the resource is served.

You can run your own facilitator for custom settlement logic. See the [x402 GitHub repository](https://github.com/coinbase/x402) for reference implementations.

## Security Notes

- **PAYMENT-SIGNATURE** headers are single-use — the `nonce` in the EIP-3009 authorization prevents replay attacks.
- The `validBefore` timestamp bounds the payment window (default 5 minutes).
- The `payTo` address in the payment requirements must match your configured `X402_PAY_TO` — clients can verify this before paying.
- Never share your wallet's private key. The receiving wallet only needs an address — no private key is stored on the server.
